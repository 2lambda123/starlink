What It Can Do

First, it can reduce spectrographic data in an automatic way. No system, however, will do everything every user will ever want so some post processing may be required in certain circumstances. The aim is to produce publishable quality spectra at the telescope via an automated reduction paradigm.

Briefly, Portable-CGS4DR can do the following:

  • Allows a wide variety of data reduction configurations.
  • Interlaces oversampled data frames.
  • Reduce known BIAS, DARK, FLAT, ARC, OBJECT and SKY frames.
  • Wavelength calibrate via suitable ARC lines.
  • Flux calibrate via a suitable STANDARD.
  • Remove the SKY, residual sky OH-lines (< 2.3 µm) and thermal emission (> 2.3 µm) from data.
  • Add data into groups for improved signal-to-noise.
  • Extract and de-ripple a spectrum.
  • Maintains an index of reduced observations.
  • Maintains, if required, an archive of observations.
  • Plot data in a variety of ways.

The basic concept of the system is based upon re-scheduling to read a data reduction queue. This queue is maintained in memory.
