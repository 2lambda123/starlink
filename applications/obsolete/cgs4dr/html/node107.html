How to Enable Overgraphs

It might appear from the other sections of this document that the set_something_display routines refer to specific ports. If that were true, there would be no way to enable overgraphs during the automated display of reduced data. How then do we overcome this restriction? The answer lies in allowing the procedure to prompt for a response and entering one of the following (where sans-serif type indicates an input):

ICL> set_observation_display
Display in port 0? > yes
Display in port 1? > yes
Display in port 2? > yes
Display in port 3? > yes
Display in port 4? > yes
Display in port 5? > overgraph port=1 cut=X spos=20 epos=40 colour=red
Display in port 6? > overgraph port=1 cut=X spos=25 epos=30 colour=blue
Display in port 7? > overgraph port=1 cut=X spos=30 epos=35 colour=green
Display in port 8? > overgraph port=1 cut=X spos=15 epos=45 colour=yellow

In this example, port 0 - 4 display data in whatever way they are set up. Ports 5 - 8, however, are used as dummies to display overgraphs in port 1 (which has presumably been set up to display graphs in the first place). Note the syntax of the overgraph command line. The parameters have the following meaning:

PORT - The port in which to display the overgraph.
CUT - cut direction (x, X, y, Y, h, H, v or V).
SPOS - Start row of slice for extraction of graph data.
EPOS - End row of slice for extraction of graph data.
COLOUR - The colour of the pen for the overgraph plot.
