Starting the Software

The Portable-CGS4DR GUI can be started with three optional command line parameters:

$1 is a data directory /${home}/
$2 is a UT date /current UT-date/
$3 is the file type DST or NDF /ndf/

To start the software, source the Starlink login and cshrc files and use your local data directory. E.g.:

% source /star/etc/login
% source /star/etc/cshrc
% cgs4dr_tcl /scratch/pnd/19940811 940811 ndf

If any of the command line parameters are missing, a confirmation dialogue box is created. Items labelled in red script within this box are those that require confirmation.

Once started, as a background job, the window shown in figure 11.3 will appear. This has a red background to indicate `stop' whilst the tasks are loaded. As each task becomes available, the appropriate traffic light will change to green. When all four tasks are `go', the window disappears to be replaced by 3 full-size windows (cred4, p4_disp and qman) and two iconic windows (red4 and p4 control). At this point, Portable-CGS4DR using TCLADAM is ready.

There is also a `setup' facility for defining environmental variables without starting the software:

% source /star/etc/login
% source /star/etc/cshrc
% cgs4dr_setup /scratch/pnd/19940811 940811 ndf
Setup Portable-CGS4DR V1.3-0 for /scratch/pnd/19940811
%
