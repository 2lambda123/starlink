enter_int_range

ICL> enter_int_range p1 p2 p3 p4
Use: Recursively writes ``REDUCE iyymmdd_p1_n'' (for n = p2 to p3) to database for pre-defined UT-date
p1 = Observation number /1/
p2 = Start integration number /1/
p3 = End integration number /p2/
p4 = Queue position /`oldest'/
E.g: enter_int_range 1 1 5 newest
