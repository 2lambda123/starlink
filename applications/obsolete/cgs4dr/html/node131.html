The Reduction Task Interface

The interface to the reduction task monolith, RED4, is shown in figure 12.0. The interface is the simplest of its kind being a suite of commands contained within a listbox. An action is invoked by double clicking on a selected item using mouse button 1. Some actions are `direct' i.e they do something with no further input ( e.g. status) whilst others display dialogue boxes for manipulation of the action parameters ( e.g. add_two_images). Error and informational messages are displayed in the text pane.

Figure 12.0: The RED4 Interface

Before any such action is invoked, however, the interface checks that the task is ready to execute commands and is not busy doing other things. If the task is busy the confirmation dialogue box shown in figure 12.1 is returned. In this case, the Cancel button aborts the action whereas the OK will continue and will wait until the task becomes free.
