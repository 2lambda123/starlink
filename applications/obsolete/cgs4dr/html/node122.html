The Portable-CGS4DR Style

The Portable-CGS4DR graphical user interface provides a window to each major task. Each window consists of a title bar, a menu bar, a text pane and a wigdet tree in some order. In addtion, STARTCL also provides a GWM widget which includes the xdisplay plotting surface. Note that the style described below may change as users feedback their preferences.

It may also appear different if your ~/.Xdefaults file has been edited for any reason. In particular, if you use white or near-white as a foregound colour, the Portable-CGS4DR GUI may be unreadable. The way around this is to invoke the following command before invoking the GUI:

% xrdb < /dev/null

The defaults for the Portable-CGS4DR interface are:

OK Button
- Performs some action i.e. it does something
Cancel Button
- Aborts without action i.e. does nothing
Dismiss Button
- Dismisses an informational dialogue box i.e. does nothing
Background
- wheat
Text Pane
- snow
Scrollbars
- lightyellow (inactive)
- palegreen (active)
Checkbuttons
Used for binary choices ( e.g. autoscaling in the plotting interface).
- wheat with grey border (inactive)
- all grey (active)
Radiobuttons
Used for multiple but restricted choices ( e.g. Yes, No or Ask).
- wheat with grey border (inactive)
- palegreen with grey border (active)
Actionbuttons
Used to invoke some action either by communicating directly with the task ( e.g. Plot) or generating a dialogue box for further input ( e.g. flux_calibrate).
- pink (inactive)
- palegreen (active)
Menus
- wheat (inactive)
- palegreen (active)
Menu Items
- wheat (inactive)
- palegreen (active)
Cursor
- green arrow (normal)
- orange pirate (a dialogue box is open)
- red wrist-watch (task busy)
- yellow pencil (help box open)
Mouse Button 1
- Single Click: Invokes action e.g. OK or Cancel
- Double Click: Invokes action in listbox
Mouse Button 2
- Single Click: Invokes dynamic default if appropriate
- Double Click: Clears entry widget
Mouse Button 3
- Invokes on-line help dialogue box
