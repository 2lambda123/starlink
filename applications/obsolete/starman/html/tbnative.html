TBNATIVE -
Convert table format from different machine

This programs converts a table made on a different type of machine to the format suitable for the machine the program is run on.

This is not essential, as the programs automatically convert tables written on different machines, but it saves time and simplifies things if this is done.

The table must already be in NDF format.

1 Parameters

  • 1.1 In
  • 1.2 Out

1.1 In

(Default - null)

The name of the file with the input table.

The table must be a standard Starman table format file. It is a Starman ADAM .SDF image, with `real' 32-bit real values, with each column having a header of up to 20 characters, and each row having a name of up to 20 characters.

1.2 Out

(Default - null)

The name of the file for the output table.

The table will be a standard Starman table format file. It is a Starman ADAM .SDF image, with `real' 32-bit real values, with each column having a header of up to 20 characters, and each row having a name of up to 20 characters.
