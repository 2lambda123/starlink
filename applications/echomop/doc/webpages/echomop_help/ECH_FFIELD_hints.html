Hints

A special mode of operation is triggered by setting the parameter TUNE_FFNXPLY. If the parameter is set to zero then no modeling in the x direction will be performed. If the parameter is set to 1, then no polynomials are used (would be constant), but the balance factors are calculated using the local mean value based on a 5 pixel sample. This will normally be used when the flat field at the dekker limits cannot be modeled because its intensity changes too rapidly on a scale of 1 pixel due to undersampling of the profile.

In the situation where the user independently produces a balance factor frame, then this may be fed to ECHOMOP as the flat field frame. The parameter TUNE_PREBAL should then be set to YES. In this case the modelling will not take place and the balance factors will simply be copied from the supplied frame. This should be used for special cases where the polynomial model cannot generate appropriate values.
