Input Data Files

Summary

The package performs most of its data access via Starlink subroutine libraries. All data frames are assumed to be contained in Starlink sdf files. The data themselves being stored in a 2-dimensional array accessible via the path 

    /filename/.NDF.DATA_ARRAY

A few ASCII format files are also read by the package:

  • /arctype/.ARC - Line list files are stored in ASCII format for ease of maintenance. ECHOMOP reads the line list(s) and creates its own proprietary file format. These features databases are created by ECHOMOP itself using the ASCII format files as input.

  • ECHARC.LIS - A list of identified features and their properties. Used for data exchange with the ECHARC program.

A few container files are also read by the package:

  • /name/.sdf - These Feature databases are created by ECHOMOP using the arc line lists as input. A feature database describes the relationship of each feature (arc line) to its nearest neighbours and is used in automatic identification.
