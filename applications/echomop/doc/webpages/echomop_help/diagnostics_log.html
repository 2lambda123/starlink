Diagnostics Log

Activating the diagnostics class of event logging (See 'Event log') causes ECHOMOP to produce a very detailed trace of the operations performed during the reduction. This slows the reduction down substantially and may also occupy a fair amount of disk space. Diagnostics should therefore only be enabled when a problem with the software is suspected (or it crashes).

To create a diagnostics event log, set the TUNE_DIAGNOSE parameter to YES. Then repeat the operations which led to the problem/crash. The problem should then be reported using the command 

    ECH_BUGSWAT

which will e-mail the most recent diagnostics and normal event logs' to the package maintainer for investigation.
