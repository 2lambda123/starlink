BAD_ORDER

Usage

BAD_ORDER=_INTEGER

Description

Specify which order is to be disabled from further processing. The default is set to zero; so if a mistake is made then attempting to disable order zero will cause no damage. Orders are disabled by writing the 'bad value' quantity into the first coefficient of the trace polynomial. This ensures that they also appear disabled to any other tasks (almost all of which will need the trace info). To re-enable an order it must therefore have a polynomial re-fitted to its traced-centre samples.
