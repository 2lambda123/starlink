Bug synopsis:
   $ADAM_USER not defined by echomopsetup.

Severity:
   Usability problem

Bug description:
   The environment variable $ADAM_USER is not defined by the
   ECHOMOP setup procedure.  This appears to be used to locate
   structure files and so on.

Bug replication method:
   Just start ECHMENU...the error message appears.

Workaround:
   % setenv ADAM_USER ${HOME}/adam

   does the trick.
Status:
This will be fixed in the next fix release of ECHOMOP.
